IT HELP DESK TICKET TRACKER - BUILD PATH

Build rule:
Move in small phases. Finish, test, and document each phase before starting the next one.

Current stack:
Python, Flask, SQLite, HTML, CSS, and light JavaScript only when useful.

Current app status:
- Flask app is running with active routes for home, ticket submission, employee lookup, admin queue, admin ticket detail, ticket conversation replies, and health check.
- SQLite stores submitted tickets.
- Ticket numbers are generated automatically.
- Employees can submit tickets and look up tickets by ticket number, email, or both.
- Employee ticket number lookup shows full details, while email-only lookup shows a compact ticket list.
- Admins can view tickets in a queue, open ticket details, update triage fields, and review basic metrics.
- Admins can search and filter the ticket queue by keyword, status, priority, issue type, and assignee.
- Admins can add private internal IT notes to individual tickets.
- Admins and employees can add employee-visible ticket conversation replies.
- Smart priority suggestion is rule-based and currently used as a triage aid.
- Critical smart-suggested tickets are visually marked in the admin queue.
- The Critical priority filter includes both IT Critical and Smart Critical tickets.
- Visible timestamps are shortened to date plus hour and minute.
